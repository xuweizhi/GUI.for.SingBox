# 节点测速连续重试与错误分类设计

## 目标

提高单节点和批量测速对偶发网络故障的容错能力，并在节点卡片中展示可理解的失败分类及原始错误，避免把所有失败统一显示为“不可用”。

## 范围

- 单节点和批量测速共用连续重试逻辑。
- 每个节点最多尝试 3 次，任意一次成功后立即停止。
- 相邻失败尝试间隔 300 ms，避免短时间内密集请求。
- 三次全部失败后，根据 API 返回的错误文本分类。
- 节点卡片展示分类、尝试次数和原始错误提示。
- 批量取消后不再启动排队节点；正在执行的节点完成当前请求后停止后续重试。

本次不监听或关联 sing-box 实时日志。Clash 延迟 API 仅返回通用错误时，界面显示“测试失败”，不推测更具体的根因。

## 数据模型

新增节点测速错误分类：

- `dns`
- `authentication`
- `tls`
- `connection-refused`
- `network-unreachable`
- `connection-reset`
- `timeout`
- `unknown`

节点错误状态保存以下信息：

- 分类代码。
- 最后一条原始错误。
- 已执行尝试次数。

成功结果保存延迟及成功所用尝试次数。现有节点延迟状态 `untested`、`success`、`failed` 保持不变。

## 错误分类

错误分类器接收任意异常，先使用现有错误标准化逻辑转换为字符串，再按不区分大小写的关键词匹配。匹配顺序如下：

| 分类 | 关键词 |
| --- | --- |
| DNS 失败 | `lookup`、`no such host`、`nxdomain` |
| 认证失败 | `authentication`、`unauthorized` |
| TLS/Reality 失败 | `certificate`、`tls`、`reality` |
| 连接被拒绝 | `connection refused` |
| 网络不可达 | `network is unreachable`、`no route` |
| 连接被重置 | `reset by peer`、`eof` |
| 连接超时 | `timeout`、`deadline exceeded` |
| 测试失败 | 未匹配的其他错误 |

分类器只解释 API 提供的信息。`An error occurred in the delay test` 等通用响应归类为“测试失败”。

## 重试流程

控制器提供统一的内部节点探测流程：

1. 检查节点是否存在、是否可测速，以及是否已在测速。
2. 发起第 1 次延迟 API 请求。
3. 返回正延迟时立即成功，不再重试。
4. 请求异常或返回非正延迟时记录错误。
5. 尚未达到 3 次且未取消时等待 300 ms，再次请求。
6. 三次全部失败后分类最后一次错误，并保存总尝试次数。

返回 `delay <= 0` 时沿用 `home.nodes.unavailable`，并按未知错误显示“测试失败”。

单节点测速和批量测速都调用该流程。批量进度以节点为单位：一个节点完成全部尝试或成功后，`completed` 才增加 1。

## 取消语义

现有异步池负责阻止新的排队节点启动。控制器增加批量取消检查，使正在重试的节点在当前 API 请求结束后不再等待或发起下一次请求。

由于现有 API 请求没有独立的 `AbortSignal`，本次不强制中断已经发出的请求。取消不会把尚未完成的节点计为失败。

## 界面展示

成功状态：

- 第 1 次成功：显示现有格式，例如 `82 ms`。
- 第 2 或第 3 次成功：显示 `82 ms · 2 次` 或 `82 ms · 3 次`。

失败状态：

- 直接显示本地化分类，例如“DNS 失败”“连接超时”“认证失败”。
- 显示尝试次数，例如“连接超时 · 3/3”。
- 使用现有 tips 指令在悬停时展示最后一条原始错误。

中英文语言包均增加分类和尝试次数文案。卡片布局保持当前响应式网格，不新增弹窗或设置项。

## 测试

纯函数测试覆盖：

- 每个分类关键词。
- 大小写差异。
- 通用及未知错误降级。

控制器测试覆盖：

- 第 1 次成功时不重试。
- 第 1 次失败、第 2 次成功时停止，并记录 2 次尝试。
- 三次失败后保存分类、原始错误和 `3/3`。
- `delay <= 0` 参与重试。
- 批量测速沿用相同重试行为，统计仍按节点计算。
- 批量取消后，活动节点不再发起下一次重试。
- 后续刷新得到有效历史延迟时清除本地错误和尝试状态。

组件测试覆盖：

- 成功重试次数展示。
- 失败分类及 `3/3` 展示。
- 原始错误绑定到提示指令。
- 未测试状态保持现有展示。

## 兼容性

- 不修改 sing-box API 请求格式。
- 不修改持久化配置结构。
- 不改变现有批量并发限制。
- 不将运行时测速错误写入配置文件。
